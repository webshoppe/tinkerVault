# Contributing

Thanks for your interest in improving the Markdown Viewer. This is a small, self-contained project; there is no build server and no package registry. Contributions are made by editing the source and rebuilding the single-file output.

## Where things live

All source is in `src/`: `template.html` (structure + placeholder tokens), `app.js` (the entire application script), and `src/vendor/` (`base.css`, plus the vendored `marked.min.js` / `highlight.min.js`). The shipped `index.html` at the app root is generated by `src/assemble.py`, never edited directly; edit `template.html` or `app.js` instead. See [`src/DEV_GUIDE.md`](src/DEV_GUIDE.md) for the full pipeline and placeholder map.

## Where a new feature would slot in

Most new functionality is UI- or parse-adjacent. Here is the usual map:

- **New toolbar control** (button, toggle, input): add the element in `template.html`'s toolbar block, style it in `src/vendor/base.css`, and wire its event listener in `app.js`.
- **New rendering behavior**: the parser is `marked` and the highlighter is `highlight.js`, both vendored under `src/vendor/`. Change parsing/theme by editing `app.js`/`template.html` or swapping the vendored lib files (then rebuild).
- **New persisted setting**: extend the `store` object and the `saveState`/`loadState` helpers in `app.js` so it survives reloads via `localStorage`.
- **New export option**: the export logic builds a standalone HTML string; extend it there, keeping the output fully self-contained (inline everything, no external URLs).

After any change, rebuild with `python src/assemble.py` and open the result in a browser to verify.

## Known unverified edge cases

- **Export visual styling**: the **Export** feature produces a structurally valid, self-contained standalone HTML file (embeds the highlighted code and rendered content, closes properly), but the exported file's visual styling was not re-confirmed end-to-end in a second browser session as part of the v1.1.0 pass. When working on or near export, open an actual exported `.html` in a browser to confirm it renders as expected.
- **Cross-session `localStorage` persistence** (open tabs/edits restored after reload) was confirmed by a human smoke test rather than the automated build harness, which blocks storage access. Re-verify by opening two same-named files, reloading, and confirming both tabs return.

## Code hygiene expectations

- Keep vendored libraries as-is (don't minify or transform `marked.min.js` / `highlight.min.js` beyond how they ship).
- Don't introduce `eval()` or dynamic code execution of your own.
- Prefer readable, unobfuscated app code in `app.js`.
- Bump the `VERSION` file when you ship a change; never hand-edit the footer string in `template.html` or the built file.
